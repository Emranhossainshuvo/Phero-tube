Discuss the scope of var, let, and const

var... var is the old way to declare a variable, many people use it today but most of the people use let or const instead of var. var is a function scoped variable declaration mathod. Where ever you declare a variable with the keyword var you can access to the variable from anywhere of the .js file but if you declare it in a function then you cannot access it from outside of the function.

Tell us the use cases of null and undefined

Null is normally means that there is no value in the thing you want to access it can be a variable or can be a array or object. On the other hand Undefined means that the value you're trying to acces isn't defined yet that means un-defined. Both Null and Undefined are primitive data types of JavaScript.

Null use case: if I know that any data can be missing from user or from API then I will set a condition that if the data is missing then I will set there null.

Undefined use case: While we set null for a value or data but we woun't set undefined intentionally for a data or value. Personally, I consider undefined as an error till now.
